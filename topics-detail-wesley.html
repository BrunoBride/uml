Introdução as Fases de Desenvolvimento

As fases do desenvolvimento com UML são as etapas que seguem um ciclo de vida do sistema, desde a concepção até a manutenção.

As fases podem variar conforme o método de desenvolvimento, mas geralmente são:
análise (definir o problema e os requisitos), projeto (definir a solução e a arquitetura), implementação (codificar o sistema), teste (verificar o funcionamento e a qualidade) e manutenção (corrigir, melhorar ou adaptar o sistema).

Análise - Projeto - Implementação - Teste - Manutenção.

Fase de Análise:
Nesta fase, o objetivo é entender o problema que o sistema deve resolver, as necessidades e expectativas dos usuários, os requisitos funcionais e não funcionais, e o escopo do projeto. Nesta fase, são utilizados diagramas de caso de uso, diagramas de atividades e diagramas de estado para representar as funcionalidades, os cenários e os fluxos do sistema. Uma analogia para esta fase é a de um arquiteto que visita o terreno onde será construída uma casa, conversa com o cliente sobre o que ele quer e precisa, e faz um esboço do projeto da casa.




Fase de Projeto:
Nesta fase, o objetivo é definir a solução que o sistema irá implementar, a arquitetura, os componentes, as interfaces, os padrões e as tecnologias que serão utilizados. Nesta fase, são utilizados diagramas de classes, diagramas de componentes, diagramas de implantação e diagramas de comunicação para representar a estrutura, o comportamento e a interação dos elementos do sistema. Uma analogia para esta fase é a de um arquiteto que faz o projeto detalhado da casa, especificando os materiais, as medidas, os cômodos, as instalações e os desenhos técnicos.




Fase de Implementação:
Nesta fase, o objetivo é transformar o projeto do sistema em código, usando a linguagem de programação escolhida. Nesta fase, o desenvolvedor cria as classes, os objetos, os métodos e as propriedades que compõem o sistema, seguindo as regras e os requisitos definidos nas fases anteriores. Uma analogia para esta fase é a de um construtor que constrói a casa, usando os materiais, as ferramentas e as técnicas adequadas, de acordo com o plano, o orçamento e o cronograma do arquiteto.




Fase de Teste:
Nesta fase, o objetivo é verificar se o sistema atende aos requisitos e se funciona corretamente. Nesta fase, são utilizados diagramas de visão geral da interação, diagramas de tempo e diagramas de estado para representar os testes, as validações e as transições dos elementos do sistema. Uma analogia para esta fase é a de um inspetor que verifica se a casa está de acordo com o projeto, se não há defeitos, se tudo funciona e se o cliente está satisfeito.




Fase de Manutenção:
Nesta fase, o objetivo é corrigir, melhorar ou adaptar o sistema, de acordo com as mudanças de requisitos, de tecnologias ou de ambiente. Nesta fase, são utilizados os mesmos diagramas das fases anteriores, mas com modificações, adições ou remoções de elementos do sistema. Uma analogia para esta fase é a de um reformador que faz reparos, alterações ou ampliações na casa, de acordo com as novas necessidades, preferências ou tendências do cliente.
